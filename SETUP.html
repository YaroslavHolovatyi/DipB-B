# Local Setup — Ubuntu / Debian

Everything you need to install on your Linux laptop before writing code for Beer & Beverages. Walk through it top to bottom; the order matters because later steps depend on earlier ones.

# What you're installing and why

Tool Why we need it
Build tools, git, openssl headers Required for building Python wheels and native deps
Docker + Compose Runs PostgreSQL (with PostGIS) and Redis in containers — keeps your laptop clean
Python 3.12 + uv FastAPI backend runtime + the fastest Python package manager
PostgreSQL client (psql) Inspect the database directly from the terminal
Node.js 22 LTS via nvm Runs the Expo dev server and the build tooling
EAS CLI Builds installable .apk / .ipa artefacts for the demo
Expo Go (on your phone) Loads the dev build over the QR code without a native build
VS Code Editor with Python + ESLint + Prettier support
GitHub CLI / SSH key Pull / push to GitHub
OpenAI account + API key Powers the receipt OCR (Vision) and Tavern Tales (GPT-4o)
DBeaver (optional) GUI for inspecting the database visually
HTTPie (optional) Friendly curl alternative for hitting the API
direnv (optional) Auto-loads .env when you cd into the project

The total install time on a fresh Ubuntu is roughly 30–45 minutes including downloads.

# 1. System packages

Bring the system up to date and install the build essentials, network tools, and library headers that the rest of the stack depends on.

sudo apt update && sudo apt upgrade -y
sudo apt install -y \
    build-essential \
    git \
    curl \
    wget \
    ca-certificates \
    gnupg \
    lsb-release \
    pkg-config \
    libssl-dev \
    libffi-dev \
    libpq-dev \
    postgresql-client \
    jq \
    unzip \
    zip

postgresql-client gives you psql so you can connect to the dockerised database without installing the full Postgres server on your laptop. libpq-dev is there in case you ever need to compile a Python driver that needs it (asyncpg doesn't, but it's harmless).

# 2. Docker Engine + Compose

We use Docker to run PostgreSQL (with PostGIS) and Redis locally — no system pollution, easy to reset, identical to what we'll deploy. Install the official Docker repo (the version in Ubuntu's main repo is usually old).

# Add Docker's official GPG key and repo
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Run docker without sudo
sudo usermod -aG docker $USER
# Apply the new group right now without logging out:
newgrp docker

# Sanity check
docker run --rm hello-world
docker compose version

If you're on Debian (not Ubuntu), swap linux/ubuntu for linux/debian in both URLs.

# 3. Python 3.12 via uv

uv is a modern Python package manager (built by Astral, written in Rust). It handles virtualenvs, lockfiles, and can even install Python itself for you. Much faster than pip / poetry.

# Install uv to ~/.local/bin
curl -LsSf https://astral.sh/uv/install.sh | sh

# Reload your shell so ~/.local/bin is on PATH
source ~/.bashrc            # or ~/.zshrc

# Verify
uv --version

# Install Python 3.12 (uv keeps it under ~/.local/share/uv/python/)
uv python install 3.12
uv python list

You don't need to install Python with apt if you go this route — uv manages the runtime. Later, inside the backend/ folder we'll run uv sync and it creates a virtualenv with Python 3.12 and all dependencies in one command.

# 4. Node.js 22 LTS via nvm

We use nvm so you can switch Node versions later without sudo. Don't apt install nodejs — Ubuntu's version is too old and globally installed packages will be a mess.

# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# Reload shell so nvm is available
source ~/.bashrc            # or ~/.zshrc

# Install Node 22 LTS and make it the default
nvm install --lts
nvm alias default 'lts/*'

# Verify
node --version    # v22.x
npm --version     # 10.x or 11.x

While you're here, install pnpm (faster than npm for the Expo monorepo) and the EAS CLI for builds:

npm install -g pnpm
npm install -g eas-cli

pnpm --version
eas --version

# 5. Expo Go on your phone

You'll run the React Native app on your real phone — fastest dev loop, no emulator needed for the day-to-day.

  1. On your Android phone: open Play Store, install Expo Go.
  2. Make sure your phone and laptop are on the same Wi-Fi network.
  3. Later, when you run npx expo start from the mobile/ folder, scan the QR code with Expo Go and the app loads.

No Android SDK install required for this flow. If you want an emulator later, install Android Studio (~10 GB) and create an AVD; we'll cross that bridge when we need it.

# 6. VS Code + extensions

The whole project is editable in VS Code. Install via Microsoft's repo (the snap is fine but the deb gets updates faster):

wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -D -o root -g root -m 644 packages.microsoft.gpg /etc/apt/keyrings/packages.microsoft.gpg
sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
rm -f packages.microsoft.gpg

sudo apt update
sudo apt install -y code

Then install these extensions (open VS Code → Extensions panel, or run from the terminal):

code --install-extension ms-python.python
code --install-extension ms-python.vscode-pylance

code --install-extension esbenp.prettier-vscode
code --install-extension bradlc.vscode-tailwindcss
code --install-extension ms-azuretools.vscode-docker
code --install-extension cweijan.vscode-postgresql-client2

# 7. Git config + GitHub SSH key

Set your identity (used in every commit):

git config --global user.name  "Yaroslav Holovatyi"
git config --global user.email "yaroslavgolovatyy@gmail.com"
git config --global init.defaultBranch main
git config --global pull.rebase false

Generate an SSH key and add it to GitHub so git push doesn't ask for a password every time:

ssh-keygen -t ed25519 -C "yaroslavgolovatyy@gmail.com"
# Accept defaults, optionally set a passphrase

# Print the public key — copy this to GitHub
cat ~/.ssh/id_ed25519.pub

Then open https://github.com/settings/keys, click "New SSH key", paste the public key, save.

Optionally install GitHub's CLI for gh repo create, gh pr, etc.:

type -p curl >/dev/null || sudo apt install -y curl
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | \
    sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | \
    sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install -y gh
gh auth login

# 8. OpenAI account + API key

Tavern Tales (GPT-4o) and receipt OCR (Vision) both call OpenAI. You need an account with billing.

  1. Go to https://platform.openai.com/signup and create an account.
  2. Add a payment method at https://platform.openai.com/settings/organization/billing/overview — you need to top up at least $5.
  3. Create an API key at https://platform.openai.com/api-keys. Copy it somewhere safe — it's shown once.
  4. Set a monthly hard cap on the billing page (start with $10–20) so a runaway loop can't drain your wallet.
  5. Later, in backend/.env, set OPENAI_API_KEY=sk-....

For development the cost is small. A typical Tavern Tales turn is ~1500 tokens in + 400 tokens out ≈ $0.01. Receipt OCR is ~$0.005 per receipt. The per-user quota in dnd_usage_quota caps individual abuse.

# 9. Optional but useful

A database GUI — DBeaver Community Edition. Useful for poking around the schema when something looks off:

sudo snap install dbeaver-ce

HTTPie — much friendlier than curl for hitting the API while you build endpoints:

sudo apt install -y httpie
# usage: http POST localhost:8000/api/auth/login email=foo@bar.com password=secret

direnv — auto-loads .envrc per directory so you never accidentally commit a key:

sudo apt install -y direnv
# Add to ~/.bashrc:
echo 'eval "$(direnv hook bash)"' >> ~/.bashrc
source ~/.bashrc

# 10. Project directory & docker-compose

The repo lives at ~/Desktop/DiplomaProject. The docker-compose.yml in the repo root brings up PostgreSQL (with PostGIS) and Redis on the standard ports. The first time you run it, the Postgres container creates the empty database; you then apply the schema.

cd ~/Desktop/DiplomaProject

# Start the services in the background
docker compose up -d

# Watch them boot
docker compose logs -f postgres

# Connect with psql to confirm
psql "postgres://app:app@localhost:5432/beer_and_beverages" -c "select version();"

# Apply the schema (one-shot for now; Alembic migrations come in Phase 1)
psql "postgres://app:app@localhost:5432/beer_and_beverages" -f database/schema.sql

# Confirm: should print 44
psql "postgres://app:app@localhost:5432/beer_and_beverages" -c "\dt" | wc -l

To wipe everything and start over:

docker compose down -v
docker compose up -d

# 11. Verify everything is installed

Run the verification script from the repo root. It prints the version of each tool and fails loudly if anything is missing:

bash scripts/verify-setup.sh

Expected output is something like:

✓ git           2.43.0
✓ docker        27.x
✓ docker compose v2.x
✓ uv            0.5.x
✓ Python        3.12.x
✓ node          v22.x
✓ npm           10.x
✓ pnpm          9.x
✓ eas-cli       14.x
✓ psql          16.x
✓ code          1.95.x

If any line is missing, scroll back to the step that installs that tool.

# 12. Next steps after setup

Once the verify script is clean, you're ready to start Phase 1.

  1. Initialise the backend: cd backend && uv init && uv add fastapi uvicorn[standard] sqlalchemy[asyncio] asyncpg alembic pydantic-settings python-jose[cryptography] passlib[argon2] redis openai.
  2. Initialise the mobile app: cd mobile && pnpm create expo-app . --template. Then add Tamagui, Redux Toolkit, RTK Query, and the WebSocket client.
  3. Run alembic init alembic inside backend/, point it at database/schema.sql, and generate the first migration.
  4. Start the dev servers: uv run uvicorn app.main:app --reload in one terminal, cd mobile && npx expo start --tunnel in another.
  5. Open Expo Go on your phone, scan the QR code, log in.

That's Phase 1 of the roadmap. From there each phase ends with something demoable.
